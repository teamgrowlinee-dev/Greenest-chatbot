Ülevaade

Greenest chatbot widget on retseptiassistent, mis aitab teie klientidel leida retsepte ja lisada vajalikke tooteid ostukorvi. Widget on kirjutatud puhtas JavaScriptis ja ei sõltu ühelegi raamistikult – sobib igasse veebilehte olenemata tehnoloogiast (Nuxt, Next.js, Laravel, staatilised HTML-lehed jne).

Arhitektuur:

  • Widget JS/CSS – laetakse Growlinee Render serverist (https://greenest-chatbot.onrender.com)
  • Retseptid ja tooted – päritakse Google Apps Scriptist, andmed Google Sheetsis
  • Ostukorv – ühendatakse teie saidiga cartAdapter kaudu (vt jaotis 3)

1. Widget'i paigaldamine (embed)

Meetod A — Üherealine loader (soovitatav)

Lisage oma veebilehe <head> sektsiooni üks rida:

<script src="https://greenest-chatbot.onrender.com/widget/loader.js" defer></script>

See laadib automaatselt nii CSS-i kui ka widget JS-i. Enne seda rida saate lisada oma konfiguratsiooniobjekti:

<script>
  window.GREENEST_WIDGET_CONFIG = {
    // Teie saidile omased seaded (ülekirjutavad vaikeväärtused)
    vendor: "greenest",
    recipeOnlyMode: true,
    defaultServings: 2,
    storeLocale: "et",           // "et" või "en"
    cartAdapterMode: "hybrid",   // kasutab automaatselt teie cart API-t

    // Vajadusel: custom cart adapter (vt jaotis 3)
    // cartAdapter: { ... }
  };
</script>
<script src="https://greenest-chatbot.onrender.com/widget/loader.js" defer></script>

2. Toodete integratsioon

Widget kuvab tooteid retseptide koostisosadena. Toodete andmed pärinevad Google Sheetsi AI_catalog lehelt. Selleks et widget töötaks korrektselt teie tegeliku poe toodetega, peavad täidetud olema järgmised tingimused:

2.1 Toote ID peab kattuma teie cart API-ga

Kui kasutaja lisab retsepti ostukorvi, saadab widget teie cart API-le toote ID selliselt:

POST https://api.greenest.ee/api/cart/item/add
Body: { "product_id": "114", "quantity": 1 }

Nõue: AI_catalog lehel olev product_id veerg peab sisaldama täpselt samu ID-sid, mida teie api.greenest.ee/api/cart/ tunnistab.

2.2 AI_catalog lehe struktuur

Google Sheetsi lehe AI_catalog veerud peavad olema järgmised:

2.3 Toodete kataloogi hoidmine ajakohasena

Praegu uuendatakse kataloogi käsitsi. Soovitame kahte lähenemist:

  • Käsitsi eksport: Exportige tooted CSV-na oma süsteemist ja importige Google Sheetsisse regulaarselt (nt iganädalaselt)
  • Automaatne sünkroniseerimine: Kui teie backendil on toodete API endpoint, saame ehitada n8n automatisatsiooni, mis sünkroniseerib muutused automaatselt (Growlinee saab selle seadistada)
Tähtis: Kui toote ID AI_catalog lehel ei kattu cart API ID-ga, ei lisata toodet ostukorvi. Kontrollige alati, et ID-d on täpselt samad (sama formaat – nt "114" mitte 114).

3. Ostukorvi integratsioon

3.1 Automaatne adapter (soovitatav – ei nõua arendust)

Widget sisaldab sisseehitatud adapterit, mis tunneb automaatselt ära greenest.ee domeeni ja kasutab teie cart API-t otse:

// Widget kutsub automaatselt neid endpoint'e:
GET  https://api.greenest.ee/api/cart/content/token   → saab sessiooni tokeni
POST https://api.greenest.ee/api/cart/item/add        → lisab ühe toote
POST https://api.greenest.ee/api/cart/item/add-many   → lisab mitu toodet korraga
GET  https://api.greenest.ee/api/cart/content/list    → pärib korvi sisu

Selleks et automaatne adapter töötaks, peavad need endpoint'id:

  • Olema kättesaadavad https://api.greenest.ee/api/cart/ aadressil
  • Toetama Cart-Session-Token header-it autentimiseks
  • Tagastama JSON vastused (vt allpool)
  • Lubama CORS päringuid greenest.ee domeenilt

3.2 Endpoint'ide oodatav formaat

// GET /api/cart/content/token
// Vastus:
{ "data": { "token": "abc123xyz", "expires_in": 7776000 } }

// POST /api/cart/item/add
// Päring:
{ "product_id": "114", "quantity": 1 }
// Vastus (edukad):
{ "ok": true } // või mis iganes teie API tagastab

// GET /api/cart/content/list
// Vastus:
{
  "data": [
    { "product_id": "114", "name": "Pasta", "quantity": 2, "price": 3.88 },
    ...
  ]
}

3.3 Custom adapter (kui vajate erilist käitumist)

Kui soovite täpsemalt kontrollida, mis juhtub ostukorvi lisamisel, kasutage custom adapterit:

<script>
  window.GREENEST_WIDGET_CONFIG = {
    // ... muud seaded ...
    cartAdapterMode: "external",  // lülitab automaatse adapteri välja
    cartAdapter: {

      // Kutsutakse kui kasutaja vajutab "Lisa ostukorvi"
      // items = tootemassiiv: [{ product_id, name, qty_base, unit, price, ... }]
      // context = { sessionId, recipeId, servings, clientId }
      addItems: function(items, context) {
        items.forEach(function(item) {
          // Teie enda ostukorvi loogika
          myStore.cart.add(item.product_id, item.qty_base);
        });
      },

      // Kutsutakse kui kasutaja vajutab korvi ikooni
      openCart: function(items, context) {
        myStore.cart.open();
      },

      // Peab tagastama checkout URL-i stringina
      getCheckoutUrl: function(items, context) {
        return "https://greenest.ee/et/checkout?session=" + context.sessionId;
      }
    }
  };
</script>
Soovitatav lähenemine: Kasutage cartAdapterMode: "hybrid" koos cartAdapter objektiga. Sellisel juhul kasutab widget teie adapterit kui see on olemas, muidu langeb tagasi automaatsele adapterile.

4. CORS konfiguratsioon

Widget teeb brauserist päringuid kolmele aadressile. Kontrollige, et teie server lubab need:

Päringute sihtkohtKellelt peab tulemaMida teha?
script.google.com (Apps Script) Brauserist Ei nõua muudatusi – Google haldab CORS-i ise
api.greenest.ee/api/cart/ greenest.ee domeenilt ✅ Peate lubama CORS päringud greenest.ee ja www.greenest.ee domeenilt
greenest-chatbot.onrender.com Brauserist (staatiline) Ei nõua muudatusi – meie server lubab kõiki päringuid

CORS header näide (teie backend)

// Lubage need päritolud:
Access-Control-Allow-Origin: https://greenest.ee
Access-Control-Allow-Origin: https://www.greenest.ee
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Content-Type, Cart-Session-Token, App-Language

5. Kiire testleht (arenduseks)

Testimiseks looge ajutine HTML-fail ja avage brauseris:

<!DOCTYPE html>
<html lang="et">
<head>
  <meta charset="UTF-8">
  <title>Greenest Widget Test</title>
  <script>
    window.GREENEST_WIDGET_CONFIG = {
      vendor: "greenest",
      recipeOnlyMode: true,
      cartAdapterMode: "hybrid",
      enableChatLogging: true,
      debug: true,  // lisab konsooli logid
      cartAdapter: {
        addItems: function(items, ctx) {
          console.log("Lisa korvi:", items, ctx);
          alert("Korvi lisatud: " + items.length + " toodet");
        },
        openCart: function() { console.log("Ava korv"); },
        getCheckoutUrl: function() { return "https://greenest.ee/et/checkout"; }
      }
    };
  </script>
  <script src="https://greenest-chatbot.onrender.com/widget/loader.js" defer></script>
</head>
<body>
  <h1>Test leht</h1>
  <p>Widget peaks ilmuma paremas allnurgas.</p>
</body>
</html>
