Greenest Chatbot Widget
Developer Integration Guide

Prepared by: Growlinee  |  Version: 1.0  |  Date: February 2026  |  Contact: team.growlinee@gmail.com

Overview

The Greenest chatbot widget is a recipe assistant that helps your customers find recipes and add the required products to their cart. The widget is written in plain JavaScript with no framework dependencies — it works in any website regardless of technology stack (Nuxt, Next.js, Laravel, static HTML, etc.).

Architecture:

  • Widget JS/CSS – served from the Growlinee Render server (https://greenest-chatbot.onrender.com)
  • Recipes and products – fetched from Google Apps Script, data stored in Google Sheets
  • Cart – connected to your site via the cartAdapter interface (see section 3)
⚠️ Critical requirement before integrating: Product IDs in the AI_catalog Google Sheet must exactly match the IDs that your cart API (api.greenest.ee/api/cart/item/add) accepts. If the IDs don't match, the recipe will appear to be added to the cart visually, but the actual cart API call will fail. Verify this before going live (see section 2).

1. Widget installation (embed)

Method A — Single-line loader (recommended)

Add one line to your website's <head> section:

<script src="https://greenest-chatbot.onrender.com/widget/loader.js" defer></script>

This automatically loads both the CSS and the widget JS. Optionally add your configuration object before this line:

<script>
  window.GREENEST_WIDGET_CONFIG = {
    // Your site-specific settings (override the defaults)
    vendor: "greenest",
    recipeOnlyMode: true,
    defaultServings: 2,
    storeLocale: "et",           // "et" or "en"
    cartAdapterMode: "hybrid",   // automatically uses your cart API

    // Optional: custom cart adapter (see section 3)
    // cartAdapter: { ... }
  };
</script>
<script src="https://greenest-chatbot.onrender.com/widget/loader.js" defer></script>

Method B — Manual file references

<!-- CSS -->
<link rel="stylesheet" href="https://greenest-chatbot.onrender.com/widget/greenest-widget.css" />

<script>

2. Product integration

The widget displays products as recipe ingredients. Product data comes from the AI_catalog sheet in Google Sheets. For the widget to work correctly with your store's actual products, the following conditions must be met:

2.1 Product ID must match your cart API

When a user adds a recipe to the cart, the widget sends the product ID to your cart API like this:

POST https://api.greenest.ee/api/cart/item/add
Body: { "product_id": "114", "quantity": 1 }

Requirement: The product_id column in the AI_catalog sheet must contain exactly the same IDs that your api.greenest.ee/api/cart/ recognises.

2.2 AI_catalog sheet structure

The AI_catalog Google Sheet must have the following columns:

2.3 Keeping the product catalogue up to date

The catalogue is currently updated manually. We recommend one of two approaches:

  • Manual export: Export your products as a CSV from your system and import into Google Sheets regularly (e.g. weekly)
  • Automatic sync: If your backend has a products API endpoint, we can build an n8n automation that syncs changes automatically (Growlinee can set this up)
Important: If a product ID in AI_catalog does not match the cart API ID, the product will not be added to the cart. Always verify that IDs are exactly the same — including format (e.g. "114" not 114).

3. Cart integration

3.1 Auto adapter (recommended — no development required)

The widget has a built-in adapter that automatically detects the greenest.ee domain and calls your cart API directly:

// The widget automatically calls these endpoints:
GET  https://api.greenest.ee/api/cart/content/token   → get session token
POST https://api.greenest.ee/api/cart/item/add        → add single item
POST https://api.greenest.ee/api/cart/item/add-many   → add multiple items at once
GET  https://api.greenest.ee/api/cart/content/list    → get cart contents

For the auto adapter to work, these endpoints must:

  • Be available at https://api.greenest.ee/api/cart/
  • Support the Cart-Session-Token header for authentication
  • Return JSON responses (see below)
  • Allow CORS requests from the greenest.ee domain

3.2 Expected endpoint response format

// GET /api/cart/content/token
// Response:
{ "data": { "token": "abc123xyz", "expires_in": 7776000 } }

// POST /api/cart/item/add
// Request:
{ "product_id": "114", "quantity": 1 }
// Response (success):
{ "ok": true } // or whatever your API returns

// GET /api/cart/content/list
// Response:
{
  "data": [
    { "product_id": "114", "name": "Pasta", "quantity": 2, "price": 3.88 },
    ...
  ]
}

3.3 Custom adapter (if you need custom behaviour)

If you want more control over what happens when items are added to the cart, use a custom adapter:

<script>
  window.GREENEST_WIDGET_CONFIG = {
    // ... other settings ...
    cartAdapterMode: "external",  // disables the auto adapter
    cartAdapter: {

      // Called when the user clicks "Add to cart"
      // items = product array: [{ product_id, name, qty_base, unit, price, ... }]
      // context = { sessionId, recipeId, servings, clientId }
      addItems: function(items, context) {
        items.forEach(function(item) {
          // Your own cart logic
          myStore.cart.add(item.product_id, item.qty_base);
        });
      },

      // Called when the user clicks the cart icon
      openCart: function(items, context) {
        myStore.cart.open();
      },

      // Must return a checkout URL as a string
      getCheckoutUrl: function(items, context) {
        return "https://greenest.ee/en/checkout?session=" + context.sessionId;
      }
    }
  };
</script>
Recommended approach: Use cartAdapterMode: "hybrid" together with a cartAdapter object. The widget will use your adapter when provided, and fall back to the auto adapter otherwise.

4. CORS configuration

The widget makes browser requests to three addresses. Verify that your server allows these:

Request targetOriginAction required
script.google.com (Apps Script) Browser No changes needed — Google manages CORS itself
api.greenest.ee/api/cart/ greenest.ee domain ✅ You must allow CORS requests from greenest.ee and www.greenest.ee
greenest-chatbot.onrender.com Browser (static) No changes needed — our server allows all origins

CORS header example (your backend)

// Allow these origins:
Access-Control-Allow-Origin: https://greenest.ee
Access-Control-Allow-Origin: https://www.greenest.ee
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Content-Type, Cart-Session-Token, App-Language

</body>
</html>
